# 页面布局设计

## 1. 全局布局 (GamePage)

游戏主界面使用响应式栅格将导航、主体与辅助面板并列展示，顶部 Header 内嵌 `ActionQueue`。桌面端包含背包预览列，移动端仅保留导航 + 主体两栏。

### 1.1 桌面端布局 (≥1024px)

```text
+---------------------------------------------------------------------+
| Header (Game Title + ActionQueue + global controls)                 |
+---------------------+---------------------------+-------------------+
| Left Sidebar        | Main Content (RouterView) | Inventory Preview |
|                     |                           |   (Desktop only)  |
+---------------------+---------------------------+-------------------+
```

- **Header**: 贯穿整页上方，呈现游戏标题与 `ActionQueue`，并留出全局操作空间。
- **LeftSidebar**: 展示所有技能、宝箱与战斗入口，包含等级进度条与分组标签。
- **Inventory 预览列**: 桌面端常驻的背包侧栏，可浏览与交互物品；宽度可由用户拖拽分隔条进行调整。

### 1.2 移动端布局 (<1024px)

```text
+-----------------------------------------------------------+
| Header (ActionQueue inline, title可选隐藏)                 |
+-------+---------------------------------------------------+
| Nav   | Main Content (RouterView)                         |
+-------+---------------------------------------------------+
```

- **LeftSidebar**: 收敛为窄列，仅显示技能简称与必要图标。
- **Header**: 保留 `ActionQueue` 与关键按钮，其他元素根据空间选择性隐藏。
- **Inventory**: 预览列折叠，玩家通过独立的背包页面查看与管理物品。

## 2. 页面特定布局

### 2.1 技能页面 (SkillPage)

```text
+-----------------------------------------------------------+
| Gradient Stat Card (name / level / description / progress) |
+-----------------------------------------------------------+
| Optional Tab Row (per skill tab groups)                   |
+-----------------------------------------------------------+
| Action Button Grid (square entries -> detail modal)       |
+-----------------------------------------------------------+
| Fixed Consumable Bar (production skills only)             |
+-----------------------------------------------------------+
```

- 顶部信息卡突出技能名称、等级与进度，并提供描述文本。
- Tab 区域用于切换不同动作分组；若技能无分组则隐藏。
- 动作按钮网格呈方形入口，点击后打开动作详情弹窗。
- 生产类技能在页面底部固定展示消耗品槽位，方便随时更换。

### 2.2 战斗页面 (CombatPage)

战斗页面布局较为复杂，包含多个标签页、模态框与实时战斗状态展示。请参阅专门文档 [`ui/COMBAT_LAYOUT.md`](./COMBAT_LAYOUT.md) 获取完整细节和交互说明。

## 3. NotificationCenter 布局

NotificationCenter 是独立的浮层组件，常驻在页面右上角（移动端根据空间可适度向下偏移）。它承载所有 Toast/消息卡片，不占据栅格布局空间，出现时以堆栈方式从上到下排列。
